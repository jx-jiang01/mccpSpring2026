Section 1: Introduction ~900 words
Move 1: Establishing Territory
Centrality Claims: LLM-based agents with "extensive capabilities" for "wide range of tasks"
"In recent years, substantial efforts have been dedicated to developing agents based on large language models (LLMs), aiming at simulating human behavior to enhance the performance across a wide range of tasks."
Strong centrality claims establishing LLM-based agents as important paradigm. Connects technical capabilities to practical applications.
Move 2: Establishing Niche
Gaps Identified: Separate optimization of agents vs. collaborative feedback loop
"Existing research primarily focuses on optimizing either the recommendation agent or the user agent separately, overlooking the critical role of the feedback loop between the user and the recommender."
Gap is highly specific and researchable. The problem is framed as missing collaborative interaction rather than component deficiency.
Move 3: Occupying Niche
Solution: AFL (Agentic Feedback Loop) modeling with quantified improvements
"To this end, we introduce the Agentic Feedback Loop (AFL) modeling. AFL simultaneously constructs both a recommendation agent and a user agent, using textual communication to simulate the feedback loop."
Strong positioning with acronym introduction (AFL) and clear contribution claims. Emphasis on "simultaneously" improving both agents.
↓
Section 2: Related Work ~700 words
Move 1: Thematic Overview
Scope: LLMs for recommendation (feature enhancement, direct recommendation) and agent-based recommendation
"Due to LLMs' extensive knowledge and strong reasoning capabilities, researchers have begun exploring their application in recommendation systems."
Clear positioning relative to existing work. Two main categories identified (feature enhancement, direct recommendation).
Move 2: Critical Analysis
Themes: Recommendation agents (RecMind, MACRec), user agents (Agent4Rec, RecLLM, RecAgent)
"The recommendation agent focuses on tackling recommendation tasks... The user agent focuses on simulating user behavior... However, the existing recommendation agent and user agent approaches have not considered the feedback loop between them."
Strong critical analysis with clear limitations identified. Systematic review of both agent types reveals the gap.
Move 3: Gap Synthesis
Gap: Missing feedback loop between recommendation and user agents
Literature review concludes by explicitly identifying the feedback loop as missing element, directly motivating AFL.
↓
Section 3: Method ~1800 words
Overview & Architecture
Components: Recommendation Agent, User Agent, Feedback Loop
"As shown in Figure 2, AFL consists of: (1) a recommendation agent (Section §3.2), (2) a user agent (Section §3.3), and (3) the feedback loop between the two agents (Section §3.4)."
Highly technical section with strong architectural foundation. Clear separation of components with dedicated subsections.
Formal Problem Statement
Input-Output: Interaction history [I₁, I₂, ..., Iₙ] → Next item Iₙ₊₁
"The input to AFL is the user-item interaction history, formalized as [I₁, I₂, ⋯, Iₙ], where Iᵢ represents the i-th item the user interacts with. The output is AFL's prediction of the next item, Iₙ₊₁, that the user is likely to interact with."
Mathematical formalism appropriate for CS audience. Clear specification of task and components.
Feedback Loop Mechanism
Process: Iterative recommendation → feedback → refinement cycle
"In each iteration, the recommendation agent suggests an item and provides a rationale. The user agent then responds with feedback, indicating whether it likes the item. If the user agent is satisfied, the loop terminates; otherwise, both the recommendation rationale and user feedback are stored in memory, and the process repeats."
Detailed algorithmic description with clear termination conditions and memory management.
↓
Section 4: Experiments ~2200 words
RQ1: Effectiveness of AFL
Evaluation: Recommendation performance (HitRatio@1) and user simulation (Precision/Recall/F1)
"Based on the experimental results presented in Table 4, we have drawn the following findings and conclusions: (1) First, the recommendation agent equipped with the recommendation model outperforms or performs equally well compared to the original base model in most cases. (2) Moreover, the AFL, which is based on the feedback loop, demonstrates a significantly greater improvement than the use of a standalone recommendation agent."
Dual-task evaluation demonstrates simultaneous improvement in both recommendation and user simulation. Comprehensive comparison across multiple base models.
RQ2: Impact of Key Components
Ablation Studies: User agent feedback, recommendation/reward models, iteration number
"Extensive experiments validate the effectiveness of the proposed AFL approach, underscoring the potential and importance of the feedback loop in agent-based recommendation systems."
Systematic component analysis demonstrating importance of each element. Comprehensive ablation studies.
RQ3: Biases in Feedback Loop
Analysis: Popularity bias and position bias investigation
"Furthermore, unlike real-world feedback loops that often amplify popularity and position biases, the experimental results show that AFL does not exacerbate these biases, demonstrating its robustness."
Strong robustness claims with explicit bias analysis. Demonstrates thorough experimental design considering potential negative consequences.
↓
Section 5: Conclusion and Future Work ~400 words
Synthesis & Impact
Contributions: First feedback loop framework, AFL architecture, comprehensive validation
Results: 11.52% improvement over single recommendation agent, 21.12% over single user agent
"Extensive experiments on three datasets demonstrate the effectiveness of the agentic feedback loop: the agentic feedback loop yields an average improvement of 11.52% over the single recommendation agent and 21.12% over the single user agent."
Strong conclusion with quantified contributions connecting to broader field implications. Robustness claims (bias analysis) strengthen contribution.

Imitation Framework for Future Papers

Structural Elements to Adapt:

  • Modular Architecture: Clear separation of components with dedicated subsections
  • Feedback Loop Mechanism: Iterative refinement through agent interaction
  • Memory Management: Explicit handling of interaction history for both agents
  • Dual-Task Framework: Simultaneous optimization of recommendation and user simulation
  • Bias Analysis: Systematic investigation of fairness and robustness

Rhetorical Strategies:

  • Collaboration Emphasis: "Simultaneously" enhancing multiple components
  • Gap Identification: Missing interaction rather than component deficiency
  • Real-World Connection: Feedback loops as fundamental system feature
  • Quantitative Positioning: Specific performance improvements in introduction
  • Robustness Claims: Bias analysis demonstrating system reliability

Quality Indicators:

  • Architectural Rigor: Clear formalization of agent components and interactions
  • Empirical Validation: Multiple datasets, metrics, and baselines
  • Comprehensive Ablation: Systematic component analysis
  • Bias Investigation: Explicit fairness and robustness analysis
  • Reproducibility: Detailed implementation and publicly available code
  • Generalizability: Validation across multiple base models and domains
